# 闪念式APP使用演示脚本

## 演示场景：创业灵感记录与分析

### 场景一：突发灵感记录
**描述**：用户正在咖啡厅休息，突然想到一个创业点子

**操作流程**：
1. 打开"闪念式"APP
2. 点击右下角的"+"按钮
3. 输入想法："开发一个AI驱动的健身指导APP，根据用户体态自动调整训练计划"
4. 选择天蓝色主题
5. 保存想法

**界面展示**：
- 主界面显示刚添加的想法卡片
- 时间显示为"刚刚"
- AI分析按钮处于可用状态

### 场景二：AI深度分析
**描述**：用户想要深入了解这个想法的可行性

**操作流程**：
1. 点击想法卡片进入详情页
2. 点击"AI分析"按钮
3. 等待AI分析（显示"AI正在思考中..."）
4. 查看AI分析结果，包括：
   - 市场分析
   - 技术实现建议
   - 潜在挑战
   - 发展方向

### 场景三：添加更多想法
**描述**：基于刚才的分析，用户又有了新的想法

**操作流程**：
1. 返回主界面
2. 添加第二个想法："可以考虑先做一个MVP版本，专注于体态检测功能"
3. 添加第三个想法："可以和智能镜子厂商合作，整合硬件设备"
4. 选择不同的颜色进行区分

### 场景四：闪念网络视图
**描述**：用户想要查看想法之间的关联

**操作流程**：
1. 点击顶部网络图标
2. 进入闪念网络视图
3. 看到三个想法节点以圆形布局展示
4. AI自动绘制了连接线，显示想法间的关联：
   - 想法1 → 想法2：实现路径
   - 想法1 → 想法3：商业模式
   - 想法2 → 想法3：功能扩展

### 场景五：设置与个性化
**描述**：用户想要自定义应用体验

**操作流程**：
1. 点击顶部设置图标
2. 进入设置页面
3. 修改AI模型为"deepseek-R1"（深度思考模式）
4. 启用纸张质感效果
5. 保存设置

### 场景六：想法延伸
**描述**：AI基于现有想法提供更多创意

**操作流程**：
1. 返回主界面
2. 点击第一个想法
3. 在详情页查看AI分析结果
4. AI提出：
   - 可以加入AR技术，实时显示训练效果
   - 考虑企业版，为健身房提供解决方案
   - 与可穿戴设备集成，收集更多健康数据

### 场景七：搜索历史想法
**描述**：用户想要查找之前相关的想法

**操作流程**：
1. 在主界面长按某个想法
2. 或者通过搜索框输入"健身"
3. 显示所有包含关键词的想法
4. 搜索结果高亮显示匹配文本

### 场景八：分享想法
**描述**：用户想要将想法分享给合作伙伴

**操作流程**：
1. 点击想法卡片
2. 在详情页点击分享按钮
3. 选择分享方式（微信、邮件等）
4. 分享格式包含想法内容和AI分析结果

## 演示要点

### UI特色
1. **纸张质感设计**：米白色背景，细微纸纹纹理
2. **纯色主题**：天蓝色、科技蓝等多种颜色选择
3. **简洁布局**：卡片式设计，信息层次清晰
4. **动画效果**：流畅的过渡动画，提升用户体验

### AI功能展示
1. **深度思考**：使用deepseek-R1模型展示思考过程
2. **智能连接**：自动识别想法间的关联
3. **创意扩展**：基于现有想法生成新的创意点
4. **实用分析**：提供具体可行的建议

### 核心价值
1. **即时记录**：不错过任何灵感
2. **智能分析**：让想法更有价值
3. **可视化网络**：发现创意间的联系
4. **持续迭代**：不断丰富和扩展想法
